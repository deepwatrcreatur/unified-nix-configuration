# Design: Distributed Nix Builds for Homelab

Generated by /office-hours on 2026-03-24
Branch: main
Repo: unified-nix-configuration
Status: DRAFT
Mode: Builder

## Problem Statement

We want to use idle compute capacity from Proxmox VMs in the homelab to speed up Nix builds. Currently, builds run locally which can be slow on resource-constrained machines.

## Goals

1. Distribute Nix derivations across all available homelab builders
2. Share built packages via a common cache
3. Implement sensible garbage collection for the shared cache
4. Keep architecture simple (no Kubernetes)

## Constraints

- Proxmox cluster with multiple VMs
- CephFS available for shared storage
- Already running attic-cache host (can serve as binary cache)

## Premises

1. **No Kubernetes needed** — Standalone VMs with Nix daemon are sufficient
2. **Attic-cache as shared cache** — Use existing attic-cache host for binary cache
3. **Per-VM local stores** — Each builder has local store, shared via attic
4. **CephFS not needed for store** — Binary cache is better than shared filesystem

**Agree/disagree?**

## Implementation

### 1. Builder VM Configuration Template

Create a module that all builder VMs import:

```nix
# modules/nixos/nix-builder.nix
{ config, lib, pkgs, ... }:

{
  options = with lib; {
    nixBuilder = types.submodule {
      options = {
        enable = mkEnableOption "Enable this VM as a Nix build machine";
        maxJobs = mkOption {
          type = types.int;
          default = 4;
          description = "Maximum concurrent build jobs";
        };
        supportedFeatures = mkOption {
          type = types.listOf types.str;
          default = [ "big-parallel" ];
          description = "Nix supported features";
        };
      };
    };
  };

  config = lib.mkIf config.nixBuilder.enable {
    # Create dedicated build user
    users.users.nixbuilder = {
      isSystemUser = true;
      group = "nixbuilder";
      useDefaultShell = true;
      description = "Nix remote build user";
    };

    users.groups.nixbuilder = {};

    # Nix daemon configuration
    nix = {
      distributedBuilds = true;
      nrBuildUsers = 64;
      settings = {
        trusted-users = [ "nixbuilder" ];
        max-jobs = toString config.nixBuilder.maxJobs;
        cores = 0;  # Let Nix decide based on available cores
      };
    };

    # SSH access for remote builders
    services.openssh = {
      enable = true;
      settings = {
        PermitRootLogin = "no";
        PasswordAuthentication = false;
      };
    };
  };
}
```

### 2. Host-Specific Configuration

Each Proxmox VM gets its own config:

```nix
# hosts/proxmox-builder-1/default.nix
{ inputs, lib, pkgs, ... }:

{

    ../modules/nixos/nix-builder.nix

  nixBuilder = {
    enable = true;
```

### 3. Distributed Builds Client Config

On machines that should use remote builders:

```nix
# In flake.nix or common module
nix = {
  distributedBuilds = true;
  settings.builders-use-substitutes = true;
  settings.extra-substituters = [
    "https://attic-cache:${toString config.services.attic.settings.http.port}"
  ];
  settings.extra-trusted-public-keys = [
    "attic-cache:+..."
  ];
};

nix.buildMachines = [
  {
    hostName = "proxmox-builder-1";
    sshUser = "nixbuilder";
    sshKey = "/root/.ssh/nix-builder";
    system = "x86_64-linux";
    maxJobs = 4;
    speedFactor = 1;
    supportedFeatures = [ "big-parallel" ];
  }
  # ... add more builders
];
```

### 4. Attic Cache Configuration

Already running attic-cache, configure it to accept pushes:

```nix
# hosts/attic-cache/default.nix
services.attic = {
  enable = true;
  settings = {
    # Already configured for HTTP
    http = {
      port = 8092;
      # Allow push from builder VMs
    };
    #預
    cache = {
      # No auto-delete — we manage GC manually
    };
  };
};
```

### 5. Garbage Collection Strategy

**Option A: Cache-server GC (Recommended)**
- Run GC on attic-cache host periodically
- Keep last 30 days of builds
- Or keep last 100GB, oldest evicted first

```nix
# Attic GC schedule
systemd.services.attic-gc = {
  description = "Attic cache garbage collection";
  serviceConfig = {
    Type = "oneshot";
    ExecStart = "${pkgs.attic}/bin/attic gc attic-cache --keep-last 30";
  };
  timerConfig = {
    OnCalendar = "weekly";
    Persistent = true;
  };
};
```

**Option B: Per-builder GC**
- Each builder manages local store GC
- Runs after builds complete

```nix
# In builder VM config
systemd.services.nix-gc = {
  description = "Nix garbage collection";
  serviceConfig = {
    Type = "oneshot";
    ExecStart = "${pkgs.nix}/bin/nix-store --gc --print-dead";
  };
  timerConfig = {
    OnCalendar = "daily";
  };
};
```

## SSH Key Setup

1. Generate key on each client machine:
   ```bash
   ssh-keygen -t ed25519 -f /root/.ssh/nix-builder -N ""
   ```

2. Add public key to each builder's `authorized_keys` for `nixbuilder` user

3. Add hosts to `known_hosts` (use `ssh-keyscan`)

## Why Not CephFS for Nix Store

1. **Nix store isn't designed for concurrent access** — The SQLite database in `/nix/var/nix/db` doesn't handle multiple writers
2. **Binary cache is the standard solution** — Attic/Cachix handle this properly
3. **CephFS better for other things** — Home directories, Docker volumes, etc.

If you want shared storage for builds, use Attic as a binary cache instead.

## CephFS Use Cases (What it's good for)

- Shared home directories
- Docker/Container storage
- Media files
- Backups
