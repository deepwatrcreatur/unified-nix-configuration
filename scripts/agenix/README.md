# Agenix Migration Scripts

These scripts facilitate the migration from sops-nix to agenix.

## Scripts

### 1. `collect-host-keys.sh`
Collects SSH host keys from all configured NixOS hosts.

```bash
./collect-host-keys.sh
```

Connects to each host and saves `/etc/ssh/ssh_host_ed25519_key.pub` to `ssh-keys/{hostname}-host-ed25519.pub`.

These host keys are legacy agenix recipients during migration. Prefer dedicated
machine identities for machine-scoped secrets and store their public keys in
`ssh-keys/agenix-machine-identities/{hostname}.pub`.

**Hosts configured:**
- gateway
- workstation  
- homeserver
- attic-cache
- rustdesk

Edit the script to add/remove hosts.

### 2. `generate-secrets-nix.sh`
Auto-generates `secrets.nix` from collected SSH keys.

```bash
./generate-secrets-nix.sh
```

Creates `secrets.nix` at repo root with:
- Host keys (from `*-host-ed25519.pub` files)
- User keys (from `username@*-ed25519.pub` files)
- Secret definitions with appropriate publicKeys
- Machine identity recipients when `ssh-keys/agenix-machine-identities/{hostname}.pub` exists

**Review and customize** the generated `secrets.nix` to match your actual secrets.

## Usage Workflow

1. **Collect host keys:**
   ```bash
   cd scripts/agenix
   ./collect-host-keys.sh
   ```

2. **Generate secrets.nix:**
   ```bash
   ./generate-secrets-nix.sh
   ```

3. **Review and edit** `../../secrets.nix`:
   - Adjust secret definitions
   - Add/remove secrets as needed
   - Verify publicKeys assignments

4. **Follow migration plan** in `docs/agenix-migration-plan.md`

## Key Naming Conventions

### Host Keys
Pattern: `{hostname}-host-ed25519.pub`
- Example: `gateway-host-ed25519.pub`
- Used for: Legacy fallback recipients for system-level secrets

### Machine Identity Keys
Pattern: `{hostname}.pub`
- Example: `gateway.pub`
- Used for: Stable machine-scoped agenix recipients

### User Keys  
Pattern: `{username}@{hostname}-ed25519.pub`
- Example: `deepwatrcreatur@gateway-ed25519.pub`
- Used for: User-level secrets (home-manager)

The generator extracts unique usernames and uses the first key found per user.

## Notes

- Scripts require SSH access to all hosts
- Host keys are automatically generated by NixOS on first boot
- User keys come from `~/.ssh/id_ed25519.pub` on each host
- Generated `secrets.nix` is a starting point - customize for your needs
